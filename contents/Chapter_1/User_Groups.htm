User Groups

User Group is used to configure a group of users containing common features and common User Roles for Apps.

 

This will make the work much more efficient as you do not need to add User Roles to each and every User when you register Users in User App.

 

When registering a User in User App, you need to select the relevant User Group for the User.

 

Once a User Group is selected for a User when the User is being configured, the features in the selected User Group such as dashboard, home page etc. and the User Roles in the User Group will be applied to the User.

 

Users in the same User Group will have the same features of the User Group.

Configure a New User Group

Configuring a new User Group needs to be done in several steps.

 

 Path:

Settings icon   Configure Section  User Groups Page

 

1.   Add New User Group

 

User Groups page, click the Add New User Group link.New User Group configuration page will appear.

 

In the User Group box, type the name of the User Group that you want to configure.

 

Click Add to create the User Group. Created User Group’s detailed page will be displayed.

2.   Add Basic Details of User Group

  

User Group detail page  Basic Details tab

 

 After adding a new User Group, you need to add its Basic details;

  1. Basic Details
    1. On the Basic Details tab, hover over Basic Details text and click the Edit icon   that appears. Created Group Name will be displayed.
    2. Show App Menu check box will be selected by default. If you do not want to show the App menu to the Users with the User Group, clear the Show App Menu check box.   App menu : This is the menu displayed on the left, containing the App icons of the Apps, which are enabled to the respective User Group. It allows users to navigate to each App.
    3. In the Home Page box, type the web address of the web site that you want to have as the Home page of the Users belong to the User Group, when they log into the system.
    4. Click Save.
  2. Menu
    1. Hover over the Menu section and click the respective Edit icon  .
    2. Click the Show me an example link to see an example for Menu format.
    3. Edit accordingly and click Save.
  3. Expire Password
    1. Hover over the Expire Password section and click the respective Edit icon. 
    2. If you need the Passwords of the Users in the User Group to expire after the given time (Time is configured under Password Configurations section on App Configuration page), select the Expire Password check box and click Save 

 Setting under App Configuration

If the Expire Password checkbox is selected, the User Passwords of the Users in the User Group will expire after the given time under the Password Configuration section on App Configuration Page (Settings  Manage  App Configuration Password Configuration).

3.   Allow access to Apps via User Roles
 

User Group detail page  User Roles tab

 

Prerequisite: Required User Roles (configure under Manage  User Roles).

 

This is used to allow the User Group to access required Apps and their features according to your requirement. Accessibility to an App is provided by applying a pre-configured User Role to the respective App.

 

Note: Accessibility to the features and functionalities of an App depends on the User Role you have assigned to the App.

 

Initially, after a User Group is created, none of the Apps will be enabled/accessible to the Users in the Group.  

Grant Access to Each App Manually

To grant access to an App,

 

  1. Select the required application from Unassigned Apps and click the Add icon . A window will appear for you to select the required User Role for the respective application.
  2. Under Role list box, select the required User Role.
  3. Click Update. App name and its assigned User Role will appear under User Roles section on the User Roles tab.
  4. To add more user roles to enable more Apps for the User Group, repeat the steps 1 to 3.
  5. To change the existing User Role of an App, click the respective Edit icon.
  6. To remove an App from the assigned App list, click the respective Delete icon  . Respective App will be unassigned from the User Group.

Copy User Roles from an exisiting User Group

This allows you to copy all the configured User Roles for Apps from an existing User Group and add to the current User Group.

 

To copy User Roles from an existing User Group;

  1. Click the Copy from an existing User Group link.
  2. In the User Group box, select the User Group you want. Once a User Group is selected, its available no. of User Roles will be displayed below the selected User Group name.
  3. Click Update. All the available User Roles of the selected User group will be added to the new User Group.
  4. If you want to delete any User Role for an App, click the respective Delete icon 
  5. To edit any User Role, click the respective Edit icon , do the changes and click Update.
4. Add Dashboard Settings of User Group

 

User Group detail page  Dashboard tab

 

This allows you to configure the dashboard of the Users belong to the respective User Group. Dashboard will be arranged with the weblets of the Apps in the system.

Default Dashboard

Edit Default Dashboard

 

Edit Default Dashboard link is used to edit the Default Dashboard features.

Default Dashboard will be applied to the iviva Dashboard (the dashboard is the first page users see, when they are logged into the system. You can go to the Dashboard by clicking on the Dashboard icon   on the App menu.)

  1. Click the Edit Default Dashboard link. Existing Dashboard with its weblets will appear.
  2. Click the Edit link on the top left corner. By default the weblets of all the Apps will be shown.
  3. If you want to view weblets of any specific App, select the App name from the Show weblets for list box. E.g. User
  4. To add a Weblet to the Default Dashboard, click the respective Add to dashboard button on the selected Weblet. Added Weblet will be displayed below.
  5. Add all the required weblets that you want to see on the Default Dashboard.
  6. To remove a Weblet from the Default Dashboard, click the Delete icon   on the respective added Weblet.
  7. To re-arrange the weblets on the Default Dashboard, drag and drop the required selected weblets in the required order.
  8. To customize the appearance of a Weblet, click the respective   icon and select the required color.
  9. Click the Save link on the top right corner to save the configuration.

Custom Dashboard

You can configure a Custom Dashboard for the users of a User Group. If you enable the Custom Dashboard, the Default Dashboard will not be applied.

To configure a Custom Dashboard;

  1. To add a new Custom Dashboard
    1. Click Enable Custom Dashboards
    2. Click the Add icon . A page to add a new Custom Dashboard will appear.
    3. In the Name box, type the name of the new Custom dashboard that you want to configure.
    4. Click Add. Dashboard name will be added under Custom Dashboards section.
  2. To edit Custom Dashboard;
    1. Click Edit dashboard link displayed alongside the Custom Dashboard name. Dashboard configuration page will appear.
    2. Click the Edit link.
    3. Select the required weblets from Apps as you require. Arrange the dashboard as required and click Save.
  3. To edit the name of a dashboard, click the respective Edit icon , change the name and click Update.
  4. To delete a configured Custom Dashboard, click the respective Delete icon .

Attention –If you have any Custom dashboards, the Default dashboard will not be used. To apply the Default dashboard again, you need to click Disable Custom Dashboards button.

Application Dashboard

Application Dashboard is the Dashboard containing weblets, appear under the Dashboard tab of each App. Each App can have its own Application Dashboard. App dashboards can be configured for each User Group as required.

  1. Under the Dashboard tab of User Group detail page, click Configure Application Dashboard link.
  2. Select the App name for which you need to create its App Dashboard. (To configure the App Dashboard of User App, select ‘User’ from the Choose an app List box.)
  3. Click Configure default dashboard link.
  4. Click Edit link.
  5. Select and add the required weblets you want to have on the selected App’s Dashboard.
  6. Click Save.
5. Add Users to User Group

 

User Group detail page  Users tab

  1. To add a User to the User Group;
    1. On the Users tab of the User Group details page, click the Add icon  . User Search page appears.
    2. Click the required User name you want to add to the User Group. User will be added to the User Group.

Edit User Group

  1. On the User Groups page, search for the required user group and click to open relevant detail page.
  2. Edit the User Group details accordingly.

Delete User Group

  1. On the User Groups page, search for the required User Group and click to open relevant detail page.
  2. Click the respective Delete this User Group link or the Delete Icon  on the Side bar on the right.
  3. A message will appear to confirm the deletion. Click Delete to confirm.
  4. If the respective user group is in use by the system, deletion will not be allowed.
6. Configure UMS Dashboard Settings

 

User Group detail page  UMS Dashboard Settings

 

Note – UMS Dashboard Settings tab will be available only if Utility Management App is enabled. .

 

This feature (on the detailed page of User Group) allows you to configure the Map settings of User Groups, which will be reflected on the UMS dashboard of the Users in the respective User Group.

 

If you configure UMS Dashboard settings on User Group Detail page, those settings will override the Default Map Settings configured in UMS App (on App Configuration page) and will be applied as the UMS Dashboard settings of the Users in the respective User Group(s).

 

The Map Settings along with the Location configured here will be taken as the Location/Map settings for UMS App Dashboard of the Users belong to the respective User Group(s).

 

Preferred Locations: This section is used to add the Locations of which energy consumption is required to be monitored on UMS App Dashboard. Users of the respective User Group will be able to monitor the energy consumption of only the selected Locations under Preferred Locations section.

 

Tip – Only the Locations tied with Meter Groups in UMS App will be listed down in the Location box on the Add Preferred Locations window.
