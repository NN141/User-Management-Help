1) User Roles

This section describes how to add and manage User Roles, which give User Rights to associated Users to access functionalities in User App.

 

A User Role defines user rights/permission for app users (who have the respective User Role) to perform a collection of tasks.

 

User Rights will be added to each User Role.

 

User Rights allow users (with the respective User Role) to perform actions with regard to the User app, according to the selected/added User Rights.

 

User Rights are grouped into four ‘User Rights Groups’ to improve usability (Guest User, Normal User, Power User and Administrator) based on the typical access permission levels.

 

E.g. Administrator User Rights –

 

group the User Rights typically associated with Admins such as Master data configurations, Guest User Rights –group the User Rights which allow to view search pages/reports in app.

 

E.g. User Role - e.g. Administrator

 

User Rights – Administrator User Rights (If “Administrator User Rights” is selected, all the User rights (Application access permissions) under “Administrator User Rights” will be applied.

 

Or else you can select individual User Rights considering the permissions to each functionality in the App you need to provide your User Role.
