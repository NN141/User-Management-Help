App Configuration

App Configuration in User App, determines the App settings and Meta data used in the User Management Application.

 

  1. Click the Settings icon  and select the App Configuration link under the Manage section. App Configuration page will open.
  2. Select the relevant Scope for User Visibility under User Visibility Scope section, if it is required to restrict the visibility of users’ data to the logged-in users. User Visibility Scope: This determines the extent of Users Data in the system allowed to be viewed by the logged in User is allowed to view.   e.g. “Permission User Group Scope” allows only to view the data of the Users in the respective User Group.
  3. Select the relevant scope applied to manage Users in User App, under User Management Scope section. If you need to restrict the User Management area which can be controlled by the logged-in users, you can select the relevant User Management Scope.   e.g “Permission User Group Scope” allows only to manage the Users in the respective User Group.
  4. Select the Login ID Custom Prefix under Custom Prefix section. The prefix specified here will be used as the prefix of the User IDs generated by the system for the User accounts (e.g. Vendor User) created without user specified User IDs. e.g. If Custom Prefix is “US”, User ID shall be “US20171019021”

    • Tip – Some users such as Vendors/ Visitors are not required to login to iviva. Therefore, User ID will not be specified when their accounts are configured in Apps such as Facility Management/Property Management.

       The system will generate User IDs for them using the Prefix specified under “Login ID Custom Prefix” followed by Date + Sequential No

  1. Under the Default section, you can set the default data to be applied when a new User is registered.

Tip – Individual Users are allowed to change the default Date/Time/Currency formats described above and configure their own Date/Time/Currency Formats under their own Profile Settings.

 
  1. Date Format: This is the Date format applied by default under a new user.
  2. Time Format: This is the Time format applied by default under a new user.
  3. Currency Format: This is the Currency format applied by default under a new user.
  4. Allow users to manage password: Select this checkbox to enable the users to create their own passwords once their accounts have been configured by an Admin. Clear the checkbox to disable the option, where the Admin enters the initial password to the user account when the user account is being configured.
  5. Enable User Blacklist: This checkbox enables the User Blacklisting functionality in iviva. If this checkbox is ticked the buttons, links related to user blacklisting will be enabled.
  1. Configure notification message templates to send under Notifications section.
    1. Password Reset Notification: This will be sent to the user when a password reset request is submitted to the system.
    2. Password Register Notification: This will be emailed to user once the “Request to register new password” link (on the side bar of User Details tab) is clicked by the Admin/ Authorized user, after registering the respective user.

Tip – Once the newly registered user logged into the system and register the initial password, the “Request to register new password” link will not be displayed to the Admin. Hence this message template will be a onetime notification for a User.

  1. Password Configurations  
    1. Under ‘Minimum Password Length’ field, enter the minimum number of characters required for the password of users.
    2. Under ‘Password expires in’ field, enter the number of days that the password will be in active.

Note – For more details on Password Management standards in iviva, please refer to the Password Management Policy User Guide.
