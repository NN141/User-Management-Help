User Registration

Back to Operations Related to Registered Users Diagram
 

In User Management application, User registration is the main and the fundamental task. This is used to register each user in the system.

 

Note – Make sure that you have configured the Prerequisites and required Master Data as explained in the Chapter 1, prior to start registering Users.

Register a New User

This section explains how to configure a new User in the system using the User Management application.

 

Attention – User App’s New User Registration wizard allows configuring Users which belong to both Custom User Types and Default User Types. The wizard contains multiple tabs and the content and the tabs appear change dynamically based on the selected ‘User Type’.

 

e.g. If the selected User Type = Staff, a tab named Staff Details will appear in wizard.

 

To register a new User in the system;

  • Click the Settings icon   on the Home page and select New User under Register section. Or click New User button on the Home page. New User Registration wizard will appear.
  • User Details tab
    1. In the First Name and Last Name boxes, type the first and last names of the User accordingly.
    2. Type E-mail, Mobile and Office Phone in the respective boxes.
    3. In the Site list, select user’s Site.
    4. Select the relevant Custom User Type from the given User Type list (with only the Custom User Types, e.g. Engineer).
    5. Select the relevant User Group.

      6. Tip – For ‘Custom User Type’ configuration and ‘User Group’ configuration please refer to “User Types” and “User Groups” sections under ‘Configuration of Prerequisites’.

    6. Click Next>>>.
  • Login Details tab
    1. Login required – By default ‘Login required’ check box will be selected, which makes the User ID and Password fields enable. This is required if the User to be registered needs to login to the system. This ‘Login required’ option enables ‘User ID’ and ‘Password’ fields. Authorized personnel/admin who registers the new User needs to enter a ‘User ID’ for the new User and a ‘Password’ for the new User to login initially. This User ID and the Password will be used by the new user to login to the system (Later the User can change the password if required).
    2. Allow user to manage password – By default this checkbox is deselected. If you need to let the new user reset his/her own password from the initial login, select this ‘Allow user to manage password’ checkbox. Once selected, the ‘Password’ and the ‘Re-Type Password’ fields will be hidden and an option to send the Password Reset Notification to the user will appear.

      Clear the checkbox to disable the option, where the Admin enters the initial password to the user account when the user account is being configured.

    3. In the User ID box, type a unique ID for the User.
    4. In the Password box, type the password and re-enter the password in the Re-Type Password box. (These fields will be available if the ‘Login required’ checkbox is selected and the ‘Allow user to manage the password’ checkbox is not selected only. The Authorized user who registers the new user shall click the Generate New Password link to generate new password and display on respective fields.

      5. Note – Your password should match the following criteria of Password Policy: • Password length should be between 8-16 characters. • Password should contain at least one numeric. • Password should contain at least one special character (alphanumeric). • Password should contain at least one Upper-case letter.

      Tip –  Click this to view the entered password.  Click this to hide the entered password.  Click this icon to view the quick info on Password policy criteria.

    5. In the Identification Number box, type a unique number to identify the respective User. This number will be used to validate and figure out required users in the system in User blacklisting.

      6. Note – If User Blacklisting function is enabled (Enable User Blacklist checkbox on the General tab of App Configuration page is selected), Identification Number will be mandatory in New User Registration wizard.

      If the Enable User Blacklist checkbox in App Configuration is not selected, Identification Number field will be optional.

    6. Click Next>>>. You will be directed to the next tab which is specific to the selected User Type. E.g. If you have selected “Landlord User” as the User Type on the User Details tab, ‘Landlord User Details’ tab will be displayed. This is an optional tab, which appears based on the selected User Type. 
    7. Optional tabs will appear based on the selected User Type. Fill the Optional tab and click Next>>>. You will be directed to the Optional Settings tab.
     If the selected User Type on   the User Details tab is; Optional Tab Name  Fields

    Tenant User

    Tenant User Details

    Select the Tenant.

    Landlord User

    Landlord User Details

    Select the Landlord.

    Caterer User

    Caterer User Details

    Select the Caterer.

    Vendor User

    Vendor User Details

    Select the Vendor.

    Select the Technician checkbox to make the respective user a Technician.

    Staff

    Staff User Details

    Department: Department that the user belongs to

    Work Location: Location where the user works

    Report to: To whom that the user reports to

    Employee Number: Employee No of the Staff user

    Designation: Title/Designation of the staff user

    VIP : Select this to make the User a VIP user

    Technician: to make the User a technician
  • Other Settings tab
    1. Display Settings: Select the ‘Date Format’, ‘Time Format’ and ‘Currency Format’ accordingly. Default formats will be selected automatically. These formats will be applied to Date, Time and Currency fields respectively which appear in the applications.
    2. Select the language that the User prefers to use the applications
    3. Notification Settings:
      • If the respective User needs to be notified via e-mail, select the Receive messages by email check box.
      • If the respective User needs to be notified via short messages/texts, select the Receive messages as texts check box.
      • To notify the respective User using both emails and texts, select both check boxes.
    4. Click Register

      5.  Setting under App Configuration

      The “Date, Time and Currency Format” displayed here will be based on the configuration done under the Default Section on the App Configuration Page(Settings  Manage  App Configuration Password Configuration).

      Note – Why do we need to configure Date Format/Time Format/Currency Format? All dates, times and currencies in all the apps of the system used by the respective user will be in the “Date Format”, “Time Format” and “Currency Format” respectively, which are configured here.

    • Once the Register button is clicked new user will be registered in the system and a message will be displayed stating that user is registered. A link named View Details which opens the respective Users detailed page with relevant tabs will be diaplyed. Another link named ‘Send Welcome Message’ will be displayed as well. Once this is clicked an email will be sent to the respective User with the Welcome message and instructions.
    • Click the View Details link. Respective User’s detailed page will appear with User Details tab, User Roles tab, etc. Other tabs will be populated according to the User type of the User and other enabled Apps. (e.g. If User Type is Technician, Service Technician tab will be displayed, If the Space Management App is enabled, Seat Details tab will be displayed.)

    Tip – For more details on configuring and editing other details of User, please refer Edit User section.

    Note – User Details and User Roles tabs are common for all Users. Other tabs will appear according to the User Type and the enabled other Apps. The “User Roles” tab displays the respective User’s User Roles.

    After a new user is registered, the “User Roles” tab will be empty till relevant User Roles are configured.
