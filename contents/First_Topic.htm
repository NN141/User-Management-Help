Introduction

Application Overview

User Management application is used to define and control which types of users can utilize applications in the system and their features and how permissions are given to them to control access to features in applications.

 

This guide is mostly designed for Administrators to provide necessary information required in User Management, granting access control to each user in the system, designing reports and etc.

 

User Management Application in a Nutshell

  • User Management application is a base app in iviva, which is used for managing system users in the entire system.
  • User Management is critical as it is used for maintaining a secure system without ineffective user and access management.
  • The User Management App is mostly utilized by Administrators to configure and manage users in the system and to configure users’ access rights to applications in the system.
  • Before start configuring the System Users, the Admin shall configure Custom User Types, User Roles and User Groups.
  • User Type is a category that is used to group users according to the job or task they perform. e.g. Administrator, Technician
  • User Types are divided into two groups namely Default User Type and Custom User Types.
  • The User Types that are configured under each App fall into the Default User Type category.
  • Custom User Type is a group that categorizes the User Types that are configured by the users in User App.
  • User Role is a set of access rights granted to a subset of System Users that restrict the access to operations in each application.
  • User Group is used to configure a group containing common features and User Roles for Apps. This makes the work efficient as the admin does not need to add each and every feature and User roles to Users, when configuring them. Once the relevant User Group is selected, all features of selected will be applied to the respective User.
  • The most important operation in User Management App is configuring System Users and assigning User Roles to them and thereafter managing the users.
